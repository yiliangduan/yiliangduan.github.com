Unity Pixelshader中使用Discard的问题

在Unity的Manual中有段对GPU优化的提示:

Some GPUs, particularly ones found in mobile devices, incur a high performance overhead for alpha-testing (or use of the discard and clip operations in pixel shaders). You should replace alpha-test shaders with alpha-blended ones if possible. Where alpha-testing cannot be avoided, you should keep the overall number of visible alpha-tested pixels to a minimum.

意思是说某些类型的GPU设备，尤其是移动设备。在pixel shaders中使用了alpha-testing或者是使用discard和clip操作会导致很高的性能压力，应该使用alpha-blended来代替alpha-testing。

Unity Texture合并工具

unity 5.6.6f1

准备写一个Mesh优化的工具链，可以实现MeshRenderer的合并。要想把所有的MeshRenderer合并，则必须考虑这几种情况:

  1. 需要合并的MeshRenderer都使用了相同的Material，并且Material的属性相同。这种情况可以直接合并

  2. 需要合并的MeshRenderer使用了相同的Material，但是Material的属性存在不同，这种情况如果合并的话需要将Material的不同属性保存下来，在运行时候通过MaterialBlockProperty来改变其属性。（工具里面对这种情况暂保持不合并）

  3. 需要合并的MeshRenderer使用了不同的Material，但是Shader相同并除了Shader所使用的Texture不同之外其他属性都相同。这种情况先合并贴图，然后根据贴图再图集中的位置偏移重写MeshRenderer的UV，然后合并MeshRenderer。

  4. 需要合并的MeshRenderer使用了不同的Material并且使用Shader不同，这种情况合并不了。

第1种情况只需合并MeshRenderer操作即可，第3种情况涉及到先合并贴图再合并修改统一的Material，然后合并MeshRenderer。基于此模块化这些步骤成单独的功能：

  • 贴图合并工具
  • 重写MeshRenderer的UV的工具
  • 合并MeshRenderer工具
